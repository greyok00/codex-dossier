# Xtool iOS Shell

Dossier now includes an `xtool`-based iOS shell scaffold in `ios-xtool-shell/`.

## What Exists

- `ios-xtool-shell/` is a SwiftPM iOS app generated by `xtool`
- the shell uses `WKWebView` to load a bundled frontend build
- outgoing `http`, `https`, `mailto`, and `tel` links are handed off to the system
- `Info.plist` includes microphone and location usage descriptions for the current Dossier flow

## Web Bundle Pipeline

Build the web app for the iOS shell with:

```bash
cd frontend
npm run build:ios:shell
```

That command:

- builds the frontend with relative asset paths
- forces local mode
- writes the output bundle to `ios-xtool-shell/Sources/ios_xtool_shell/Resources/WebApp`
- copies the existing Dossier app icon into `ios-xtool-shell/Resources/AppIcon.png`

The frontend also switches to `HashRouter` automatically when running from `file://`, which keeps in-app navigation working inside the bundled shell.

## Current External Requirements

This repo work is now far enough along that the remaining blockers are external tool inputs, not missing shell code:

- a host `swift` toolchain in `PATH`
- an Apple Darwin SDK installed for SwiftPM
- an `Xcode.xip` or `Xcode.app` to feed into `xtool sdk install`

`xtool` itself makes those requirements explicit:

- it builds SwiftPM-based iOS apps
- `xtool sdk install` requires a path to `Xcode.xip` or `Xcode.app`

## Recommended Next Steps

1. Install a Linux Swift toolchain locally.
2. Obtain an `Xcode.xip` payload and run `xtool sdk install <path>`.
3. Run:

```bash
cd ios-xtool-shell
../tools/xtool/xtool-x86_64.AppImage --appimage-extract-and-run dev build --ipa
```

## Notes

- This path avoids rewriting Dossier into React Native.
- It does not remove Apple artifacts entirely; it replaces Xcode as the day-to-day build tool, but still needs Apple SDK content.
